# KRA 경주마 Top3 예측 대시보드 (Stage 04)

본 프로젝트는 한국마사회(KRA) 서울 경주 데이터를 기반으로, 특정 경주마가 **Top3 내에 입상할 확률을 예측**하고 사용자가 그 결과를 교차 검증할 수 있도록 지원하는 Streamlit 프로토타입 대시보드입니다.

## 📌 주의사항
> **본 대시보드는 경주 전 데이터 기반 Top3 진입 가능성을 분석하기 위한 시뮬레이션 목적의 프로토타입입니다. 실제 경주 결과를 보장하지 않으며, 도박 또는 베팅 권유 목적이 절대 아님을 밝힙니다. 참고용 실험 자료로만 활용해 주십시오.**

## 1. 프로젝트 개요
이 대시보드는 Stage 02~03을 거쳐 구축된 LightGBM 머신러닝 모델의 실제 분류 타당성과 유용성을 사람이 눈으로 직접 확인할 수 있도록 구축되었습니다. 
단순 인간의 휴리스틱(직관적 규칙)과 AI의 결정을 한눈에 비교하고, 어떤 팩터(Feature)가 가장 큰 영향을 행사했는지 투명하게(Explainable) 보여주는 것에 초점을 맞춥니다.

## 2. 현재 단계
* Stage 01 ~ 02: 데이터 정제 및 누수 피처 검증, 예측형 EDA
* Stage 03: 시간 기반 Split 및 LightGBM Baseline 모델 구축
* **Stage 04 (Current)**: 예측 결과 시각화 및 검증용 웹 대시보드 구현

## 3. 실행 방법
파이썬 가상환경(`uv` 등)을 활성화한 후 아래 명령어를 입력합니다.

```bash
pip install -r requirements.txt
streamlit run app.py
```
> 구동 후 출력되는 로컬 주소(보통 `http://localhost:8501`)로 브라우저에 자동 접속됩니다.

## 4. 입력 파일 (데이터 파이프라인 동기화)
본 대시보드는 데이터 용량과 파이프라인의 중복 복사를 막기 위해 **Stage 03 (kra_race_prediction_stage03_top3_modeling) 서버 경로의 산출물을 직접 참조**하게 설계되어 있습니다.
- `models/lightgbm_top3_baseline.pkl`
- `data/modeling/modeling_data_ready.csv` 등
- *만약 의존 폴더인 Stage 03 경로가 지워졌다면 앱 구동 시 경고가 발생합니다.*

## 5. 대시보드 화면 구성
대시보드는 **7개의 탭(Tab)**으로 구성되어 사용자에게 입체적인 분석 시야를 제공합니다.
1. **프로젝트 개요**: 시스템 안내 및 무보증 경고 고지
2. **경주 예측 결과**: 선택된 경주의 예측 확률치 기반 Top 3 리스트업 (`과거 운영 모드` 선택 시 실제 정답 블라인드 처리)
3. **예측 근거 설명**: 특정 말을 지정 시 Z-score 등을 바탕으로 왜 모델이 Top3로 봤는지를 초보자 친화적 문장으로 자동 해설
4. **단순 규칙 비교**: 레이팅, 평균 성적 등 단일 지표 최상위군과 LightGBM의 선택을 비교 표기
5. **모델 성능 요약**: 테스트 셋 전체의 Precision@3, Hit@3 를 타 방식과 그래프화 비교
6. **변수 중요도**: LightGBM이 의존하는 20가지 핵심 판단 기준 제시
7. **오류 분석(Error Analysis)**: 과거 테스트의 약점(거리별, 경기등급별 적중율 편차) 진단

## 6. 모델 성능 요약 (가이드라인)
| 방식 | Precision@3 | Hit@3 |
|---|---|---|
| LightGBM 모델 | 47.95% | 58.08% |
| 무작위 찍기 | 28.3% | 28.3% |
| 말 과거성적 상위 3 | 46.03% | 57.31% |

*LightGBM 방식은 인간의 단순 규칙 기반 예측율을 유의미하게 앞질렀으며, 우승마(1위마)를 예측된 3마리 내에 포함시키는 능력을 입증했습니다.*
