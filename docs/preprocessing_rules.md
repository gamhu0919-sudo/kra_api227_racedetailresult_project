# 전처리 규칙 명세서 (2025.03 - 2026.03)

## 1. 개요
본 문서는 API227 원본 XML 데이터를 분석 가능한 정형 데이터(CSV)로 변환할 때 적용된 전처리 로직 및 규칙을 설명합니다.

## 2. 규칙 상세

| 대상 컬럼 | 원본 예시 | 처리 규칙 | 처리 이유 | 주의사항 |
|-----------|-----------|-----------|-----------|-----------|
| `rcNo`, `stOrd`, `age`, `wgBudam`, `win`, `plc`, `hrRating` | `1`, `3.0`, `2.4` | `pd.to_numeric(errors='coerce')` | 수치형 분석을 위해 숫자 타입 강제 변환 | '-' 등의 문자는 NaN 처리됨 |
| `rcDate` | `20250301` | `pd.to_datetime(format='%Y%m%d')` | 시계열 연산 및 추출(월, 주차)을 위해 | 원형 문자열 보존 권장 |
| `meet` | `1` | `meet_nm` 파생 컬럼 생성 (1:서울, 2:제주, 3:부경) | 가독성 향상 및 지역 그룹화 | 원본 코드값 보존 (Key 용도) |
| `stOrd` | `1.0` | `is_winner` (1위 여부), `is_place` (3위 이내) 파생 생성 | 분류 분석(Classification) 타겟 용도 | 0(미출전) 데이터는 수집에서 제외 권장 |
| `wgHr` | `480(+2)`, `510(-12)` | `horse_weight` (현체중), `horse_weight_delta` (증감) 분리 | 체격 조건과 주력의 상관계수 분석 | 정규표현식 `(\d+)\(([\+\-]?[0-9]+)\)` 사용 |
| `rcTime` | `1.12.3` | **(참고)** 현재는 문자열 그대로 파싱되며, 추후 초 단위 변환 필요 | 속도 분석을 위한 표준화 | 경마장/거리별 포맷 차이 가능성 존재 |

## 3. 필터링 및 제외 규칙
- **기간 필터**: `rcDate`가 `20250301`에서 `20260331` 사이인 데이터만 최종 `processed` 데이터셋에 포함합니다.
- **출전 여부 필터**: `chulYn`이 `1`(출전)인 데이터만 분석 대상으로 권장합니다. (미출전마는 착순이 없어 통계 왜곡 가능성)
- **중복 제거**: `meet`, `rcDate`, `rcNo`, `chulNo` 조합의 중복이 발생할 경우 **가장 최근 수집된 행(keep='last')**을 남깁니다.

## 4. 특수값 처리 (Sentinel Values)
- `9999.9` 또는 `0.0` 등의 값이 기록에 나타날 경우, 이는 측정이 불가능한 특이 케이스(실격 등)일 가능성이 높으므로 통계 계산 시 제외하거나 별도 범주화가 필요합니다.
- 하이픈(`-`)은 전처리 단계에서 명시적으로 `NaN`으로 변환하여 통계 오류를 방지합니다.
