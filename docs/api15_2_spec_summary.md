# API15_2 (공공데이터포털 한국마사회 경주마 성적 정보) 명세 요약

## 1. 개요
* **API 명**: 한국마사회 경주마 성적 정보
* **엔드포인트**: `https://apis.data.go.kr/B551015/API15_2`
* **지원 포맷**: `XML`, `JSON` (본 파이프라인에서는 기존 시스템과의 데이터 정합성 확인과 파싱 편의성을 위하여 `JSON` 포맷으로 단일화하여 수집함.)
* **데이터 성격**: 특정 기간의 로그 데이터라기보단, **현재 시점에 등록되어 있는 말의 통산 성적과 최신 스냅샷 프로필** 정보를 표출하는 데이터 성격을 강하게 띰.

## 2. 요청 파라미터 규격 (Request Parameters)
| 필수여부 | 파라미터명 | 타입 | 설명 | 기본값 / 설정값 |
|---|---|---|---|---|
| 필수 | `ServiceKey` | String | 공공데이터포털 발급 인증키 (URL-decoded 텍스트를 파라미터로 할당 권장) | `.env` 에서 로드 |
| 선택 | `pageNo` | Integer | 조회할 페이지 번호 | 수집기에서 순회 처리 (1~N) |
| 선택 | `numOfRows` | Integer | 한 페이지 결과 수 | 통상 100~500 설정 |
| 선택 | `_type` | String | 리턴 포맷 타입 | `json` |

> *참고*: 말의 이름이나 기간 기반 과거 데이터를 추출할 수 있는 독립적 메타 파라미터(`startDate`, `hrName` 등)의 공식 지원은 빈약하거나 동작이 부정확하므로, `pageNo`를 통한 전체 스냅샷 크롤링 전략을 채택함.

## 3. 응답 구조 및 처리 방안 (Response Structure)
정상 호출 시 아래와 같은 JSON 구조가 반환됨:
```json
{
  "response": {
    "header": {
      "resultCode": "00",
      "resultMsg": "NORMAL SERVICE."
    },
    "body": {
      "items": {
        "item": [
          { "meet": "서울", "hrName": "명불허전", "hrNo": "0123456", ... }
        ]
      },
      "numOfRows": 100,
      "pageNo": 1,
      "totalCount": 8500
    }
  }
}
```

- **에러 응답**: `resultCode` 가 "00"이 아닐 경우 에러 로그를 남기고 종료하거나 재시도함. (예. `99`: 접근 권한 에러, `01`: 어플리케이션 에러)
- **페이지 처리**: `totalCount` 를 확인하여 총 조회 가능 횟수를 얻고, `totalCount / numOfRows + 1` 페이지까지 반복 수집 자동화 진행.
