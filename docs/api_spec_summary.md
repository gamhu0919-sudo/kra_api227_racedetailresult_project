# [API227] 한국마사회 경주별 상세 성적표 명세서 요약

## 1. 개요
본 문서는 한국마사회의 `racedetailresult` (경주별상세성적표) 오픈 API 스펙을 분석 관점에서 요정리한 문서입니다.

- **프로젝트 분석 대상 기간**: 2025년 3월 ~ 2026년 3월
- **기본 서비스 URL**: `https://apis.data.go.kr/B551015/racedetailresult`
- **상세 기능명**: `getracedetailresult`

## 2. 요청 파라미터 (Request Parameters)

| 파라미터명 | 타입 | 필수 여부 | 설명 | 예시 |
|------------|------|-----------|------|------|
| `serviceKey` | str | 필수 | 공공데이터포털 발급 인증키 | `4ff4...` |
| `pageNo` | int | 선택 | 페이지 번호 | `1` |
| `numOfRows` | int | 선택 | 한 페이지 결과 수 | `100` |
| `meet` | int | 필수 | 경마장 코드 (1:서울, 2:제주, 3:부경) | `1` |
| `rc_date` | str | 필수 | 경주일자 (YYYYMMDD) | `20250301` |
| `rc_no` | int | 선택 | 경주번호 | `1` |

## 4. 데이터 수집 전략 (2025.03 - 2026.03)

### 4-1. 경주번호 자동 탐색
본 프로젝트 수집기(`collect_race_detail.py`)는 각 날짜와 경마장에 대해 `rc_no`를 1번부터 순서대로 조회합니다. 만약 3회 연속 빈 응답이 오면 해당 날짜의 경주가 끝난 것으로 판단하고 다음 날짜로 넘어갑니다.

### 4-2. 페이지네이션 대응
`numOfRows=100`으로 설정하여 대부분의 경주(통상 14두 이하)가 한 페이지에 들어오도록 설계했으나, 구조적으로 페이지 루프 처리가 가능하도록 구현하였습니다.

## 5. 실데이터 검증 필요 사항
- `9999.9`: 기록 측정 불가 또는 실격 시의 센티넬 값 여부 확인
- `-` (하이픈): 결측치 또는 특정 상태(정지 등)의 표현 여부
- `rcTime`: 문자열 포맷(예: '1:12.3')의 일관성 확인 후 숫자(초) 변환 필요
- `chulYn`: 0(미출전) 데이터의 착순(`stOrd`) 처리 방식 확인
