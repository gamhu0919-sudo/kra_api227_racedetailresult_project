# Team Source 데이터 세트 병합 보고서

## 1. 개요
본 문서는 `data/team_source` 폴더에 분산되어 있던 9개의 경주 및 기수 관련 CSV 데이터를 하나의 정합성 있는 마스터 데이터셋으로 병합한 과정과 그 결과를 설명합니다. 데이터의 가용성을 높이고 향후 분석 및 모델링을 위한 깨끗한 기초 데이터를 마련하는 것이 목적입니다.

## 2. 데이터 병합 전략 및 실행 과정

### 2.1. 식별키(Key) 타입 정규화 (전처리)
수집된 데이터셋은 파일별로 기수 번호(`jkNo`), 조교사 번호(`trNo`), 마필 번호(`hrNo`), 경주 일자(`rcDate`) 항목에 대해 숫자형(`80459`)과 문자형(`080459`), 혹은 부동소수점형(`110034.0`)이 혼재되어 있었습니다. 
안전한 조인을 위해 모든 식별키를 다음 규칙에 따라 변환했습니다:
- **String 형변환**: 숫자로 인식되는 모든 식별자를 문자열로 변환
- **Trailing Zero 제거**: `.0` 형태로 저장된 실수형 꼬리표 제거
- **Leading Zero 제거**: 포맷 통일을 위해 좌측의 0 패딩을 전부 제거하여 고유 아이디 값으로 일원화.

### 2.2. 기수 성적 파일 병합 및 중복 제거
기수 성적 정보는 3개의 파일(Jeju, Seoul, Busan)로 분할되어 있었으나 컬럼 구조가 동일했습니다.
- **수행**: 3개의 파일을 수직 병합(`Concat`)
- **중복 제거**: `jkNo`를 기준으로 완전 중복되는 기수 정보를 `.drop_duplicates()`를 통해 제거
- **결과**: 기수 성적 테이블은 총 629명의 유니크한 기수 데이터로 정제 및 생성 완료.

### 2.3. 경주별 상세 성적표 중복 검증 및 결합
경주별 상세 데이터를 담고 있는 파일(7, 8, 9번 파일)은 서로 중복되거나 파편화된 데이터로 파악되었습니다.
특히 7번과 9번 파일에는 완전히 동일한 행이 636건씩 중복 생성되어 있었고, 전체 파일이 본질적으로 같은 경주 정보의 파편들로 구성되어 있었습니다.
- **수행**: 세 개의 파일을 병합한 후 식별자(`hrNo`, `jkNo`, `rcDate`)를 기준으로 중복을 완전 제거.
- **결과**: 총 87,606행 중 58,828행의 중복 데이터를 걷어내고, 고유한 **28,778행**의 깔끔한 상세 성적풀을 구성. (이는 데이터 유실이 아니라 무의미하게 중복 적재되었던 쓰레기 데이터를 걸러낸 결과입니다.)

### 2.4. 메인 성적표를 중심으로 Left Join (데이터 증식 방지)
가장 크고 중심이 되는 `1_경주성적정보(15063979).csv` (12,206행)을 기준으로 데이터를 병합했습니다.
- 1차 조인: 1번 테이블에 정제된 '기수 성적 테이블'을 `jkNo` 기준으로 `Left Join` 수행. (이때 컬럼명이 겹치는 것을 막기 위해 기수 정보 컬럼에는 `jockey_stats_` 접두어 부여)
- 2차 조인: 앞서 확보한 고유 상세 성적풀(28,778건) 중 기준 테이블에 매핑되는 부가 정보(중복 없는 컬럼)를 추출해 추가 병합.

## 3. 병합 결과 및 검증 (Validation)

- **출력 경로**: `data/processed/merged_team_source.csv`
- **기수 정보 매칭률**: 100.00% (Missing 0건)
  식별키 정규화 작업 덕분에 기준 테이블의 모든 조인키가 완벽히 매핑되었습니다.
- **데이터 증식 검증**: 병합 전 원본 베이스 파일의 12,206행이 병합 후에도 정확하게 **12,206행으로 유지**되었습니다. 다대다(M:N) 관계로 인해 행이 폭발적으로 증식하여 모수가 오염되는 리스크를 안전하게 방어했습니다.

## 4. 결론 및 다음 단계
이로써 무작위로 수집된 9개의 중복/파편 파일이 하나의 깨끗하고 안정적인 피처풀(Feature Pool)로 통합되었습니다. 
제공해드린 파이프라인(`src/05_merge_team_source.py`)은 추후 동일한 구조의 추가 데이터가 들어오더라도 안전한 병합 처리가 가능합니다. 

이제 병합 생성된 `merged_team_source.csv` 데이터를 바탕으로 상관 분석, 피처 엔지니어링 및 모델링 EDA를 더 정확하게 수행하실 수 있습니다.
